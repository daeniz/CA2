Inheritance strategy

Our solution probably going to be the same as the majorities. We decided to go with JOINED Inheritance Strategy. The goal was to make the database simple and easily readable. To add, by using, for example SINGLE_TABLE we create a table which has all the fields from both entities which wont be filled from one another, so because of that we receive a lot of nulls which are not necessary.
